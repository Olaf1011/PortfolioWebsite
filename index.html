Porting Zool to Vulkan

Core Skills: C++, Vulkan API, DX11, Renderdocs (Graphics Debugger), Working in a group, Working with a deadline, Working with client

For our third-year project, we had the awesome opportunity to port Zool Redimensioned to Vulkan. We were given the DX11 base and were left to our own accord to work together as a group and work with Sumo Digital Academy to get Zool Redimensioned to run on Vulkan. We started from scratch as Sheffield Hallam University does not teach Vulkan out of the box. We began by creating a renderer that renders a triangle to Zool Redimensioned running on Vulkan. We had to overcome many obstacles together and work as a team to finish this task within the allotted 12-week period.

Project Magma

Core Skills: C++, GLSL, Vulkan API, Renderdocs (Graphics Debugger)

Project Magma is my own renderer/framework using the Vulkan API. I created it after working on the Zool project and used my knowledge to create an improved version which I could reuse for other projects. I also created it for use in my dissertation, where I used to to create multiple different particle effects either running on the CPU or using the compute shader to run it purely on the GPU. Being made as a way for me to practice shaders. I made it hotswappable so I can quickly change out shaders and try new things.
The source code is currently private as it is still being graded for its use in my dissertation

Labyrinthia: Lost Realms

Threading and SIMD project

Core Skills: C++, SIMD, Pthreads, Semaphores, Optimising code

For project 2 of my master's degree we were tasked with optimising code in a project given to us. We were giving 3 options of techniques to chose from. SIMD, Threading, and compute shader. I choose SIMD and Threading as I already done my dissertation using the compute shader. The task was to take the code and convert it to use either techinque. For the SIMD version I managed to go from 35ms per frame to 2ms per frame and doubling the amount of object on screen as that was tited to the frame rate.
As for Threading I focused on making a better solution than my disseration using semaphores to put the threads to sleep. Creating a system that doesn't have to create new threads each update. For this version I went from 43ms per frame, to 6.3ms per frame.
